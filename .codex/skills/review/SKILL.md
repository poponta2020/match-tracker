# PR Code Review スキル

PRのレビュープロンプトを読み込み、コードレビューを実行して結果ファイルに書き込みます。

## トリガー

以下のような依頼でこのスキルが使われます：
- 「PR #355 をレビューして」
- 「レビューして PR 355」
- 「review PR 355」
- 「PR 355 の5回目のレビューをして」

## 手順

1. **レビュープロンプトを特定する**
   - ユーザーの依頼からPR番号を取得する
   - レビュー回数が指定されていない場合は、`scripts/review/output/review-prompt-pr{番号}-*.md` のうち最大番号のファイルを使う
   - レビュー回数が指定されている場合は、`scripts/review/output/review-prompt-pr{番号}-{回数}.md` を使う

2. **レビュープロンプトを読み込む**
   - 特定したプロンプトファイルの内容をすべて読み込む
   - プロンプト内の「出力先」セクションからレビュー結果の書き込み先ファイルパスを取得する

3. **コードレビューを実施する**
   - プロンプトに記載されたレビュー観点に従い、差分を分析する
   - **差分だけで判断できない場合は、プロンプト内「リポジトリ情報」のパスから実���ァイルを `Read` / `Grep` で直接確認する**
     - 変更されたメソッドの呼び出し元・呼び出し先を確認する
     - Entity/DTOのフィールド定義や既存バリデーションとの整合性を確認する
     - 仕様書・設計書と実装の齟齬がないか照合する
   - 各観点について問題の有無を判断する

4. **レビュー結果を書き込む**
   - プロンプトの「出力形式」に従ったフォーマットでレビュー結果を作成する
   - 「出力先」セクションで指定されたファイルに結果を書き込む
   - **重要**: 結果は標準出力ではなく、必ず指定されたファイルに書き込むこと

5. **完了を報告する**
   - 書き込んだファイルパスと、指摘の概要（CRITICAL/WARNING/INFOの件数）を表示する
