- **ブランチ**: {{BRANCH}}

{{REVIEW_RESULT}}

## 修正ルール

1. **CRITICAL（重大）** の指摘は必ず修正すること
2. **WARNING（警告）** の指摘は原則修正すること。修正しない場合は理由を説明すること
3. **INFO（情報）** の指摘は任意。対応するかどうか判断して、対応しない場合は理由を述べること
4. 修正後、既存のテストが通ることを確認すること
5. 修正が完了したら、変更内容のサマリーを出力すること

## 出力形式

修正完了後、以下の形式で報告してください：

```
## 修正サマリー

### 対応した指摘
- [CRITICAL] 指摘タイトル → 修正内容の概要
- [WARNING] 指摘タイトル → 修正内容の概要

### 対応しなかった指摘（あれば）
- [INFO] 指摘タイトル → 対応しない理由

### テスト結果
- 既存テスト: PASS / FAIL
- 追加テスト: あり / なし
```
