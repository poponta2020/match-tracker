# 抽選結果コピー可能テキスト出力 要件定義書

## 1. 概要

### 目的
管理者が `/lottery/results`（抽選結果画面）で、当該月の **キャンセル待ちのみ** を整形した「コピー可能テキスト」を取得できるようにする。テキストはページ内のテキストエリアに表示され、その場で自由編集が可能。「コピー」ボタンでクリップボードへコピーできる。

### 背景・動機
抽選確定直後、管理者が当選者・キャンセル待ち者をLINEグループへ手動告知している。当選者まで含めるとメッセージが長くなりすぎるため、キャンセル待ち列のみをコピーしてLINEに貼り付けたいというニーズがある。

## 2. ユーザーストーリー

- **対象ユーザー**: ADMIN（管理者）ロール
  - 補足: SUPER_ADMIN もADMINと同じくコピー領域を表示する（既存画面の権限スコープに準ずる）。
  - PLAYER ロールには本機能エリアは表示されない（既存の抽選結果一覧表示のみ）。
- **ユーザーの目的**: 月単位の抽選確定直後、キャンセル待ち列をLINEグループへ告知したい。
- **利用シナリオ**:
  1. 管理者が抽選を実行・確定する。
  2. `/lottery/results` を開き、当該月を選択する。
  3. 各セッションの内容を画面上で目視確認する。
  4. 画面下部に表示される「コピー領域（textarea + コピーボタン）」を確認する。
  5. テキストを必要に応じてその場で編集する（補足追記、誤字修正など）。
  6. 「コピー」ボタンを押し、LINEグループへ貼り付けて送信する。

## 3. 機能要件

### 3.1 表示条件

- **コピー領域の表示条件（AND）**:
  - 現在ログイン中のロールが ADMIN または SUPER_ADMIN
  - 選択中の年月・組織で抽選が **確定済** （`/api/lottery/is-confirmed` が `true` を返す）
- 上記条件を満たさない場合、コピー領域は非表示（要素自体を描画しない）。
- PLAYER ロールには常に非表示。

### 3.2 表示位置・UIレイアウト

```
[月選択 < 2026年4月 >]

[各セッション表示（既存のまま）]
  - セッション1
  - セッション2
  - …

──────────────────────────────────
[管理者向けコピー領域（条件付き表示）]
  <textarea>（自動生成テキスト、編集可能）</textarea>
  [ コピー ]ボタン
──────────────────────────────────
```

- コピー領域は **画面最下部**（各セッション表示の下）に配置する。
- textarea の高さは内容が見やすい程度（目安: 12行程度、内容に応じてスクロール）。
- 「コピー」ボタンは textarea の直下に配置。

### 3.3 出力テキストフォーマット

#### 全体構造

```
<月>月の抽選結果（抽選落ちのみ）です

<セッション1>
<セッション2>
…

ご確認のほどおねがいします。
```

- 先頭行: `<月>月の抽選結果（抽選落ちのみ）です`
  - `<月>` は表示中の月（例: `4`）。年は含めない。
- 先頭行の後に空行1行。
- 各セッション間に空行1行。
- 末尾行: `ご確認のほどおねがいします。`
  - 最後のセッションと末尾行の間に空行1行。

#### セッション部の構造

```
<M>/<D>（<曜>）<会場名>
★<N>試合目★
1. <選手名>
2. <選手名>
…
★<N+1>試合目★
（なし）
…
```

- 1行目: `<M>/<D>（<曜>）<会場名>`
  - `<M>` はゼロ埋めなしの月（例: `4`）
  - `<D>` はゼロ埋めなしの日（例: `5`）
  - `<曜>` は1文字の日本語曜日（`月` `火` `水` `木` `金` `土` `日`）
  - `<会場名>` は `LotteryResultDto.venueName`。`null` の場合は固定文言 `会場未設定` を出力。
- 試合行: `★<N>試合目★`（`<N>` はゼロ埋めなしの試合番号）
- キャンセル待ち列: `<順位>. <選手名>`
  - 順位は `waitlistNumber` の昇順
  - ステータス `WAITLISTED` のみを対象とする（`OFFERED`、`WAITLIST_DECLINED` などは除外）

#### セッション・試合の表示ルール

- **試合の表示**:
  - セッション内の全試合（`matchResults` の全エントリ）を試合番号昇順で出力する。
  - 当該試合に `WAITLISTED` ステータスのプレイヤーが0名の場合、試合行の直後に `（なし）` 1行を出力する。
- **セッションの表示**:
  - セッション内のいずれかの試合に `WAITLISTED` プレイヤーが1名以上いる場合、当該セッション全体を出力する（空試合は `（なし）` を出力）。
  - セッション内の全試合で `WAITLISTED` プレイヤーが0名の場合、当該セッションは丸ごと出力から省略する。

#### 出力例

```
4月の抽選結果（抽選落ちのみ）です

4/5（土）麹町区民館
★1試合目★
1. 田中太郎
2. 山田次郎
3. 佐藤三郎
★2試合目★
1. 鈴木史郎
2. 高橋五郎

4/12（月）麹町区民館
★1試合目★
1. 田中太郎
★2試合目★
（なし）

ご確認のほどおねがいします。
```

### 3.4 コピーボタンの挙動

- ボタンラベル: `コピー`
- クリック時:
  1. textarea の **現在の内容**（管理者が編集していればその編集後の内容）をクリップボードへコピーする。
  2. トースト等のフィードバックで「コピーしました」を表示する（既存のトースト表現があればそれを利用、なければ簡易表示で良い）。
- クリップボードAPI失敗時: アラート等で「コピーに失敗しました」と通知。

### 3.5 textarea の挙動

- **初期値生成タイミング**: 画面表示時（`fetchResults` 完了直後）に自動生成し、textarea の初期値としてセットする。
- **再生成（上書き）タイミング**:
  - 月切替（< > ボタンで月を変更したとき）→ 新しい月のテキストで上書き
  - 画面再フェッチ後（辞退/復帰操作などで `fetchResults` が走ったとき）→ 上書き
  - いずれの場合も、編集途中の内容は破棄される（確認ダイアログ等は不要）。
- **編集制限**: textarea は通常通り自由編集可能。編集内容のバリデーションは行わない。

### 3.6 月全体でキャンセル待ちが0名の場合

- 全セッションが「セッション丸ごと省略」の対象となる場合（= 月内の全試合で `WAITLISTED` が0名）：
  - **コピーボタンを非活性（disabled）にする**。
  - textarea の内容は「ヘッダー行＋末尾行のみ」の状態とする：
    ```
    4月の抽選結果（抽選落ちのみ）です

    ご確認のほどおねがいします。
    ```
  - ※ ボタンは非活性なのでコピー実行はできないが、textarea には上記が表示される。

## 4. 技術設計

### 4.1 API設計

**新規API**: なし

**利用する既存API**:
- `GET /api/lottery/results?year=<Y>&month=<M>` — 月別抽選結果取得
  - 既存レスポンス `List<LotteryResultDto>` をそのまま使用
  - 必要フィールド: `sessionDate`, `venueName`, `matchResults` 内の `matchNumber` と `waitlisted[]`（`status`, `waitlistNumber`, `playerName`）
- `GET /api/lottery/is-confirmed?year=<Y>&month=<M>&organizationId=<orgId>` — 抽選確定済判定
  - 戻り値が `true` のときのみコピー領域を表示
  - `organizationId` は ADMIN の場合 `AuthContext` から取得した自団体 ID を渡す

### 4.2 DB設計

変更なし。

### 4.3 フロントエンド設計

#### 変更対象ファイル
- `karuta-tracker-ui/src/pages/lottery/LotteryResults.jsx`

#### 追加する状態
- `isConfirmed: boolean` — 当該月の抽選確定済フラグ（`isConfirmed` API の結果）
- `copyText: string` — textarea にバインドされる現在の編集テキスト
- 月切替時・`fetchResults` 完了時に `isConfirmed` を再取得し、`copyText` を再生成する。

#### 追加するヘルパー関数
- `buildCopyText(year, month, sessions): string`
  - 入力: 表示中の年月と `LotteryResultDto[]`
  - 出力: 上記 3.3 のフォーマットに整形したテキスト
- `formatSessionDate(sessionDate): string` — `M/D（曜）` 形式へ整形（既存ユーティリティがあれば利用）
- `getJapaneseWeekday(date): string` — 日付から `月`〜`日` の1文字曜日を返す
- `hasAnyWaitlisted(sessions): boolean` — 月全体で WAITLISTED が1名以上いるか（コピーボタン disabled 判定用）

#### 追加するコンポーネント要素（同一ファイル内）
- 管理者向けコピー領域のJSXブロック
  - 条件分岐: `(role === 'ADMIN' || role === 'SUPER_ADMIN') && isConfirmed`
  - `<textarea>` + `<button>コピー</button>`
  - クリップボードAPI: `navigator.clipboard.writeText(copyText)` を使用

#### 認可・ロール判定
- `AuthContext` から `currentPlayer.role` を取得（既存の方法に従う）
- ADMIN の自団体IDも `AuthContext` から取得して `isConfirmed` API に渡す

### 4.4 バックエンド設計

変更なし。

## 5. 影響範囲

### 5.1 変更が必要な既存ファイル
- `karuta-tracker-ui/src/pages/lottery/LotteryResults.jsx` — コピー領域の追加、既存表示はそのまま

### 5.2 既存機能への影響
- **PLAYER ロールへの影響**: なし（コピー領域は ADMIN/SUPER_ADMIN のみ表示）
- **既存「抽選結果通知」機能（`/api/lottery/notify-results`）との関係**: 完全に独立した別機能。あちらはアプリ内通知＋LINE自動送信、本機能は管理者がLINEグループへ手動コピペするためのテキスト生成。
- **`/api/lottery/results` API**: レスポンス形式に変更なし、既存呼び出し元への影響なし。
- **`/api/lottery/is-confirmed` API**: 既存APIをそのまま利用、変更なし。
- **DB スキーマ**: 変更なし。
- **CLAUDE.md ルール「DBマイグレーション適用」**: 本機能では SQL 追加・変更なし。

### 5.3 ドキュメント更新
- `docs/SCREEN_LIST.md` — `/lottery/results` 画面の機能説明に「管理者向けコピーテキスト出力」を追記
- `docs/SPECIFICATION.md` — 該当機能の追加（必要に応じて）
- `docs/DESIGN.md` — フロントエンドのみの変更で API 仕様変更がないため、原則更新不要（記載があれば追記）

## 6. 設計判断の根拠

| 判断 | 理由 |
|------|------|
| 新規APIを追加せず既存APIで実現する | `LotteryResultDto` に必要な情報（`venueName`, `sessionDate`, `matchResults`, `waitlisted` 内の `status`/`waitlistNumber`/`playerName`）が揃っており、フロントエンドで整形するだけで要件を満たせる。バックエンドの増改築コストを避ける。 |
| 抽選確定済の月のみコピー領域を表示 | 「抽選確定直後の告知」というユースケースに合致。未確定のセッションが混在する状態でテキストを生成するとミス送信のリスクがあるため、`isConfirmed=true` でガードする。 |
| 月全体で待ちが0のときボタン非活性 | テキスト本文が空（ヘッダーと末尾文言のみ）になるため、誤って「対象なし」のメッセージをLINEへ送信するのを防ぐ。 |
| 月切替・再フェッチで textarea を上書き（編集破棄） | 「LINE送信前の一時的な編集」用途と限定すれば、編集保持よりも常に最新データから生成し直す方が誤送信防止に資する。確認ダイアログを挟むのも煩雑。 |
| 試合行は空でも `（なし）` を表示 | LINE受信者に「2試合目はキャンセル待ちなし」を明示的に伝えるため。試合の存在自体を省略すると「漏れ？」と誤解されるリスクを回避。 |
| セッションは全試合空のとき丸ごと省略 | LINEメッセージを短く保つため。当該セッションは告知すべき情報がないので含める意義が薄い。 |
| 出力テキストのヘッダーは月のみ（年なし） | ユーザー指定のサンプル文面に従う。年が必要な場合は管理者が textarea で編集可能。 |
| 末尾文言を固定（`ご確認のほどおねがいします。`） | ユーザー指定。編集可能なので個別調整は管理者側で対応可能。 |
