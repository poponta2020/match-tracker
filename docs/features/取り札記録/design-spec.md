# 取り札記録 デザイン仕様（design-spec）

> ロジック/データ/API/DB は [requirements.md](./requirements.md)、決まり字は [kimariji-master.md](./kimariji-master.md) を参照（重複記述しない）。

## 1. 対象と狙い
- **対象:** MatchForm（`/matches/new`・`/matches/:id/edit`）に「取り札・お手付きを記録」ブロックを追加。
- **狙い:** 出札50枚を決まり字チップで盤面に配置し、取った/取られた位置と、お手付き詳細を各自が私的に記録する。

## 2. 採用方向（A案）／不採用
- **A案（確定）:** 競技かるた盤を模した**畳表現**。札は**縦書きチップ**（実物の札風）。取った/取られたは**マス内の左右分割**（左=取った緑／右=取られた赤）。
- **不採用 B案:** 取/取られを上下積み（列全幅）。密度には強いが、縦札化で A でも7枚が収まるため左右分割を維持する A を採用。B は参考カードとして残置。
- **確定モック（canonical・最終）:** ローカル `C:/tmp/design-screen/torifuda-kiroku/torifuda-kiroku-a.html` ＋ `_torifuda-kiroku.css`。
  - Claude Design（`Match Tracker Design System` / group `取り札記録 (Redesign)`）にも push 済みだが、最終ラウンド（折りたたみ＋お手付き4種＋新記法）はユーザー指示で実装優先とし**再pushせず**。ローカルが最終。

## 3. レイアウト構成（上→下・MatchForm内）
1. 既存 chrome（緑ナビ＋試合番号タブ）・対戦相手・結果/枚数差/**お手付き回数**（この3つは常時表示のまま）。
2. **折りたたみトグル** `取り札・お手付きを記録`（`任意`＋シェブロン）。**初期は閉じ**。タップで下記を展開。
3. 【展開時】**取り札 盤面**
   - 見出し「取り札」＋**ⓘトグル**（向き・色の凡例。タップ開閉）。
   - **敵陣**（上・全幅ブリードの畳・緑の畳縁）: 3段×左右のマス。**180°回転**＝上段が手前(下)・下段が奥(上)、かつ**敵陣右＝画面の左**。
   - **不明プール**（敵陣と自陣の間）: 未配置の縦札チップ。見出しに `残り X / (50−枚数差)枚`。
   - **自陣**（下・畳）: 自分視点そのまま（上段=奥/上、自陣右=画面の右）。3段×左右。
   - 各マスは `取った(左・緑tint)｜取られた(右・赤tint)` の2分割。札チップは縦書き。
4. 【展開時】**お手付きの内容**: お手付き回数分の入力枠。各枠で種類（ひっかけ/暗記間違え/聞き間違い/その他）→種類別項目（下記）。
5. アクション（キャンセル／登録）。

## 4. コンポーネント / 主要スタイル
- 既存 MatchForm の washi パレット準拠（`_torifuda-kiroku.css`・全て `.tk` スコープ）。
- **札チップ:** `writing-mode:vertical-rl; text-orientation:upright; width:18px`。決まり字は**最大4文字**（kimariji-master 記法）なので縦に収まる。マスは中身に応じ縦伸び（`flex-wrap`・上限なし）→7枚超も可。
- **畳:** field を `margin:0 -20px` で全幅ブリード、`border-top/bottom:5px #4a6b5a`（畳縁）＋畳色グラデ。
- **取/取られ:** マス内 `.tk-take`(緑tint) / `.tk-taken`(赤tint) の2列。
- **ⓘ / 折りたたみ:** ⓘは checkbox トグルで凡例を開閉。折りたたみは同様のトグル（実装は state で可）。
- **お手付き種類別UI:** ひっかけ=`.pos4`(上段4択)／暗記=`.dir2`(方向2択・全幅)／聞き間違い=`.cardpick`(読札・触札の**100枚ピッカー**、chevron)／その他=`.ote-text`(自由記述)。

## 5. 状態
- **未展開（既定）:** 盤面・お手付き詳細は非表示。勝敗/枚数差/お手付き回数だけで登録可（=従来フロー）。
- **空:** 全札が不明プール。`残り (50−枚数差) / (50−枚数差)枚`。
- **高密度:** 1マスに多数（7枚超）→ 縦札＋マス縦伸びで収容。
- **編集:** 保存済みの `myCardPlacements`/`myOtetsukiDetails` を復元。
- **枚数差なし/指導/不明:** プール母数は50にフォールバック（requirements 3.2）。

## 6. 必要データ（詳細は requirements）
- 出札50枚: `(matchDate, matchNumber)` の札ルール→展開（nonce は DB 共有）。
- 決まり字: [kimariji-master.md](./kimariji-master.md)（フロント定数）。
- 保存: `match_card_placements` / `match_otetsuki_details`（私的）。

## 7. インタラクション / レスポンシブ
- 390px 基準。盤面は全幅ブリードで横を稼ぎ、縦札で密度に対応。横スクロールは作らない（縦伸び＋プールは縦スクロール）。
- ドラッグ&ドロップで不明⇔マスを移動（実装は簡易でも可。タップ選択のフォールバック可）。
- お手付き回数を変えると詳細枠が増減。

## 8. ガードレール準拠
- washi パレット（MatchForm準拠）／システムsans・縦札は upright／絵文字なし／緑=取った・赤=取られた・navy=主アクション。単一ライトテーマ（実アプリに合わせ意図的）。

## 9. 実装への申し送り
- 縦札の decorative 畳表現は装飾。機能は「50枚を24マス＋不明に振り分け」。
- ドラッグ&ドロップのライブラリは既存に無ければ最小自作 or タップ割当で可（実装手順書で決定）。
- 聞き間違いの札ピッカーは100枚（空札含む）から選択（横書きの小ピル表示で可）。
