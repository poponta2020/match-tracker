# 対戦履歴への読み・一人取り表示 実装手順書

対象: 選手別 月別対戦履歴 [MatchList.jsx](../../../karuta-tracker-ui/src/pages/matches/MatchList.jsx)。
方針: **フロントエンドのみで完結**（バックエンド・DB・既存API変更なし）。

## 実装タスク

### タスク1: 読み・一人取り行の取得・マージ・表示
- [x] 完了
- **概要:** 抜け番（読み・一人取り）を取得して対戦履歴リストに差し込み表示する中核実装。
  - 既存 `byeActivityAPI.getByPlayer(targetPlayerId)`（type無し）で全抜け番を取得し、`READING` / `SOLO_PICK` のみに絞る（`targetPlayerId` 変更時に取得。フィルタ変更では再取得しない）。
  - matches から `dateString -> venueName` の会場 Map を構築（同日の任意の試合の `venueName`）。
  - 抜け番を期間（`selectedYear` / `selectedMonth`）で client-side フィルタ。
  - matches と 抜け番(READING/SOLO_PICK) を統合した表示用配列を作り、`(matchDate desc, matchNumber desc)` で再ソート。各行を種別（`'match'` | `'bye'`）で描画分岐。
  - `bye` 行の表示: 「`<会場名> <N>試合目 <活動名>`」。会場が Map に無ければ会場省略。タップ無効（表示のみ）。活動アイコン（読み=BookOpen / 一人取り=User）は任意。
  - 相手前提フィルタ（結果 / 相手名検索 / 級・性別・利き手）のいずれかが有効なときは `bye` 行をレンダリング対象から除外。期間フィルタは常に連動。
  - 取得失敗時は `bye` 表示なしでフォールバック（試合履歴は従来表示、コンソールエラーのみ）。
- **変更対象ファイル:**
  - `karuta-tracker-ui/src/pages/matches/MatchList.jsx` — 取得・整形・マージ・ソート・行レンダリング・フィルタ連動を追加
  - （`karuta-tracker-ui/src/api/byeActivities.js` / `api/index.js` は既存の `byeActivityAPI` をそのまま利用。import 追加のみ）
- **依存タスク:** なし
- **対応Issue:** #866

### タスク2: 統計エリアの読み・一人取り回数表示
- [ ] 完了
- **概要:** 期間フィルタ後の抜け番から読み・一人取りの回数を集計し、統計エリアに「読み `<n>`回 ・ 一人取り `<m>`回」を活動別に併記。
  - 期間（年/月）フィルタに連動。結果/相手/属性フィルタの影響は受けない。
  - 回数が0の活動は非表示（両方0なら回数表示自体を出さない）。
  - 上部の勝敗統計（総計・級別）には含めない。
- **変更対象ファイル:**
