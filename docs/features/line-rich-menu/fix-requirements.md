# LINE リッチメニュー 改修要件定義書

## 1. 改修概要
- **対象機能**: LINE リッチメニュー（左下エリア4）
- **改修の背景**: 通知設定ボタンの利用頻度が低く、結果入力への導線を追加したい
- **改修スコープ**: リッチメニューのエリア4を「通知設定」から「結果入力」に変更

## 2. 改修内容

### 2.1 リッチメニューのアクション変更
- **現状**: エリア4がURI型アクションで `/settings/notifications` に遷移
- **修正方針**: 遷移先URIを `/matches/results` に変更（アクション型はURI型のまま）
- **修正後のあるべき姿**: ボタン押下で結果一覧画面（MatchResultsView）に遷移

### 2.2 リッチメニュー画像の更新
- **現状**: 左下ボタンに「通知設定」テキスト + メガホンアイコン
- **修正方針**: 「結果入力」テキスト + 適切なアイコンに差し替え
- **修正後のあるべき姿**: ボタンの見た目が「結果入力」として認識できる

## 3. 技術設計

### 3.1 API変更
なし

### 3.2 DB変更
なし

### 3.3 フロントエンド変更
なし（遷移先の `/matches/new` は既存画面で、全認証ユーザーがアクセス可能）

### 3.4 バックエンド変更
なし

### 3.5 リッチメニュー設定スクリプト変更
- `docs/features/line-rich-menu/setup-richmenu.sh` のエリア4のURI変更

### 3.6 リッチメニュー画像変更
- `docs/features/line-rich-menu/richmenu.png` の左下ボタン画像を更新
- 圧縮版 `richmenu_compressed.jpeg` も更新

## 4. 影響範囲
- 影響を受ける既存機能: なし（通知設定画面は他の導線からもアクセス可能）
- 破壊的変更の有無: なし
- 本番反映にはリッチメニューの再登録（setup-richmenu.sh の再実行）が必要

## 5. 設計判断の根拠
- URI型アクションを維持する理由: 条件判定はアプリ側で既に実装済みのため、postback型にする必要がない
- `/matches/results` を遷移先とする理由: 結果一覧画面から結果入力への導線がある
