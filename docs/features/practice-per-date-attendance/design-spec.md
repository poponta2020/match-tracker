# 1日分の出欠登録画面 デザイン仕様（design-spec）

> `/design-screen` の確定仕様。実装は `/define-feature practice-per-date-attendance`（技術計画）→ `/implement` で行う。
> **今回は `design-prototype.patch` を作っていない**（バックエンド＋認証を伴う実プレビュー立ち上げのコスト回避のため、実トークンを使った忠実モックで収束）。よって本ファイルがレイアウトの正典を兼ねる（§5・§7 に具体記述）。実装は既存2画面（`PracticeParticipation` / `PracticeCancelPage`）のプリミティブを流用する。

## 1. 対象と狙い
- **対象画面:** 新規ルート（1日分の出欠登録）。カレンダー `/practice` のセッション詳細ポップアップ「出欠登録」ボタンから遷移。
- **現状の不満:** 1日の出欠を直したいだけなのに、月まとめの一括入力画面（`/practice/participation`）に飛ばされ対象日を探し直す必要がある。参加は一括入力画面、キャンセルは別画面（`/practice/cancel`）と導線が割れている。
- **狙い:** 押した日付のそのセッションだけで「参加登録」と「理由付きキャンセル」を1画面で完結。
- **主ユーザー / 主な使い方:** 全ロール本人が、その日の自分の出欠を素早く確定する。

## 2. 採用した方向性
- **方向性:** 「2セクション縦分割（上=参加トグル／下=当月扱いのみ理由付きキャンセル）」＋「フラット（脱カード）デザイン」。実行ボタンは各セクション末尾のインライン（案A。下部固定バーは競合するため不採用）。
- **不採用案と理由:**
  - モーダル内完結／既存ポップアップにインライン追加 → 理由選択・オーバーレイが窮屈／既存ポップアップの改修規模が大。専用画面（遷移）を採用。
  - 下部固定バー2本（案B）→ 参加とキャンセルの両方に未確定がある瞬間の出し分けが要り複雑。インラインに。
  - 白カードの箱組み → 「カード風すぎ／AI感」。`/pairings` の脱カード方針（hairline・余白・文字階層・フラット色ラベル）に合わせて廃した。
- **確定プロトタイプ:** 忠実モック（show_widget、最終版 = `practice_per_date_attendance_mockup_v6_flat_hairline`）。patch なし。

## 3. patch に現れない設計判断
- **未参加＝参加側／参加中＝キャンセル側、の排他振り分け:** 各試合はどちらか一方のセクションにだけ出す（重複させない）。これにより「全試合参加中→キャンセルのみ」「全試合未参加→参加のみ」が自然に成立する。「参加中🔒」のロック行を参加側に併記する案は重複ゆえ廃止。
- **満員は登録をブロックしない:** 抽選前は満員でもチェック可（超過はキャンセル待ち→抽選で解決）。満員は情報表示（`満員` ラベル＋「満員でも申込できます」の注記）にとどめる。登録不可は「抽選確定済み（ステータス表示）」「伝助削除（×）」のみ。これは既存 `PracticeParticipation` の挙動そのままで、実装時に満員を誤ってブロックしないための明示事項。
- **上部バーは日付＋会場:** 一括入力画面が「◯年◯月 参加登録」なのに対し、本画面は1日固定なので「M/D(曜)　会場名」を出す方が文脈が明快。重複するヘッダーカードは廃止。会場名が長い場合はバー内で省略（ellipsis）。
- **練習会（団体）は団体カラーで強調:** 色ドット＋団体名を `org.color` で表示（カレンダーの会場名と同トーン）。複数団体日でも対象団体を一目で判別できるよう控えめにせず出す。
- **参加＝アウトライン／キャンセル＝赤塗り:** 主操作は参加だが、キャンセルは不可逆性が高いので赤塗りで重み付け。参加保存はアウトラインで軽くする。

## 4. 使用コンポーネント
- **既存プリミティブ流用:** 上部緑ナビバー（`PracticeCancelPage` 等と同型）、試合行（`PracticeParticipation` のチェック＋人数バッジ相当）、理由ラジオ（`PracticeCancelPage` の `CANCEL_REASONS` と自由記述）、`SaveProgressOverlay`（保存中/完了/エラー→「カレンダーに戻る」）、当日12時確認ダイアログ、`PlayerChip` は不要（本画面は試合行のみ）。
- **新規:** 1画面に2セクション（参加/キャンセル）を排他振り分けで同居させる構成。フラット（hairline 区切り・色付きセクション罫線・箱なし）。

## 5. 状態（state）※本ファイルがレイアウト正典
レイアウト骨子（cream `#f9f6f2` 背景、白カードの箱組みなし・hairline `0.5px #e5ddd0` 区切り）:
- **上部バー（緑 `#4a6b5a`）:** ← 戻る ＋「M/D(曜)　会場名」（白・省略あり）。
- **練習会:** 団体カラーのドット＋団体名（`org.color`、14px/medium）。
- **「参加する試合」セクション（見出し=緑 `#4a6b5a`＋下罫線）:** 参加できる（＝未参加・抽選前・伝助削除でない）試合を行表示。各行「第N試合／時間／（人数 or 満員ラベル）／チェック」。末尾に「満員でも申込できます（キャンセル待ちになる場合あり）」の注記＋右寄せ「参加を保存」（アウトライン `#a98e80`／文字 `#82655a`）。
- **「参加をキャンセル」セクション（見出し=赤 `#b3524f`＋下罫線／当月扱いのみ）:** 参加中の確定枠（WON/PENDING）を行表示（チェック）。選択後にキャンセル理由（体調不良／仕事・学業／家庭／交通機関／その他。`その他`=自由記述）。右寄せ「選択した試合をキャンセル」（赤塗り `#d9534f`）。
- **セクション出し分け（最重要）:**

  | 状態 | 参加する試合 | 参加をキャンセル |
  |---|---|---|
  | 一部参加（当月扱い） | 未参加の試合 | 参加中の試合（理由付き） |
  | 全試合参加中（当月扱い） | 非表示 | 参加中の試合のみ |
  | 全試合未参加（当月扱い） | 全試合 | 非表示 |
  | 来月扱い（抽選前の未来月） | 全試合をトグル（外す＝理由なし取消） | 常に非表示 |
  | 抽選確定済みセッション | 追加登録不可→非表示 | 当選(WON)のみ理由付き |

- **来月扱いの差分:** 参加セクションは全試合をトグルで出し、登録済みはチェック済み＝外すと理由なし取消（`registerParticipations` の全置換で反映）。キャンセルセクションは出さない。
- **操作対象外の参加（WAITLISTED/OFFERED 等）:** どちらのアクションも出さず読み取り専用の状態表示にとどめる。
- **空（データ無し）:** 両セクションとも空になる稀なケースは「操作できる試合がありません」の空表示。
- **エラー / 不明値:** 保存・キャンセルのエラーは `SaveProgressOverlay` の error 状態（状態維持で再試行）。409 は再読込。
- **プロトタイプでの再現方法:** show_widget モックはダミー（第1=参加中/第2=6名/第3=満員）で当月扱い一部参加を再現。実装時は実データで全状態を再検証。

## 6. 必要データ（すべて既存API。新規なし）
- **表示フィールド:** セッション日付・会場名・団体（`getById` / `org.color`）、試合番号・時間割（`venueSchedules`）、試合別人数（`matchParticipantCounts`）、満員判定（`matchCapacityStatuses` or capacity 比較）、伝助削除（`densukeDeletionCandidateMatchNumbers`）、自分の参加・ステータス・participantId・version・lotteryExecuted・hasAnyExecutedLotteryInMonth（`getPlayerParticipations` / `getPlayerParticipationStatus`）。
- **集計・導出:** 当月/来月判定は `resolveAttendanceMode`。当日12時判定は既存ロジック。**参加保存ペイロードは月・団体全体を積む**（`registerParticipations` が全置換のため対象日だけを送らない）。キャンセルは対象セッションの WON/PENDING participantId を集約し `cancelMultiple`。

## 7. インタラクション / レスポンシブ
- **操作:** 参加＝チェックして「参加を保存」。キャンセル＝試合選択→理由選択→「選択した試合をキャンセル」（実行前確認、当日12時以降は追加確認）。各アクション独立、完了で `SaveProgressOverlay`→「カレンダーに戻る」→`/practice`。
- **モバイル:** 375px 基準。箱なしフラット行で横スクロールなし。会場名はバー内で省略。タップ領域は行全体（ラベル領域含む）。

## 8. ガードレール準拠メモ
- 配色は既存トークン流用（緑 `#4a6b5a`／茶 `#82655a`・`#8b4513`／赤 `#d9534f`系／cream `#f9f6f2`／団体カラー `org.color`）。新しい色は発明していない。
- 日本語文言・sentence 相当のトーン。日付は `M/D(曜)`、時刻は `HH:mm–HH:mm`。
- 脱カード（hairline・余白・文字階層）は `/pairings` の既存方針に準拠。

## 9. 残課題・実装への申し送り
- ルートのパス・パラメータ名（`?sessionId=` 想定）は技術計画で確定。`/practice/:id`（PracticeDetail）と衝突しないパスにする。
- FAB ラベルを「出欠一括登録」に変更（機能不変）。ポップアップ内ボタンは「出欠登録」のまま新画面へ。
- 参加とキャンセルは同一画面だが**別アクション**（1ボタンで同時コミットしない）。両方操作したい場合は片方実行→カレンダー→再度開いてもう片方。
- **DESIGN-PROTO スタブ一覧:** なし（worktree プロトタイプ未使用のため patch/スタブなし。実装は本 spec の §5〜§7 と既存2画面を土台にゼロから配線）。
