# 組み合わせドラッグ&ドロップ 要件定義書

## 1. 概要
- **目的:** 組み合わせ画面（/pairings）の選手入れ替え操作を、ドロップダウンからドラッグ&ドロップに完全移行し、より直感的な操作を実現する
- **背景:** 現在のドロップダウン方式は、選手が多いと探すのが大変、複数の組み替えで手間がかかる、直感的でない、という課題がある。スマホがメイン利用端末であり、タッチ操作に最適化された体験が求められる

## 2. ユーザーストーリー
- **対象ユーザー:** 練習の組み合わせを管理する管理者
- **ユーザーの目的:** 対戦組み合わせを素早く直感的に変更したい
- **利用シナリオ:**
  - 自動マッチングの結果を見て、一部の組み合わせを手動で調整する
  - 待機選手を対戦に組み込む / 対戦選手を待機に回す
  - 新たな組み合わせ枠を追加して待機選手同士をペアにする

## 3. 機能要件

### 3.1 画面仕様

#### ドラッグ&ドロップ操作（編集モード時のみ有効）

**対戦組み合わせ間のスワップ**
- 選手カードを長押し（スマホ）/ クリック（PC）で掴み、別の選手カードの位置にドロップすると2人の位置が入れ替わる
- 同じペアリング内の左右入れ替えも許可（例：「山田 vs 佐藤」→「佐藤 vs 山田」）
- 異なるペアリング間のスワップも可能（例：A組の左とB組の右を入れ替え）

**待機リスト ↔ ペアリング間の移動**
- 待機リストの選手をペアリング枠にドロップ → ドロップ先の選手と入れ替え、元の選手は待機リストに移動
- ペアリング枠の選手を待機リストにドロップ → その選手が待機に回る（片方空欄の行が残る）

**新規ペアリング作成ドロップゾーン**
- 組み合わせリストの下部に「新しい組み合わせを作成」用のドロップゾーンを配置
- 待機リストから選手を1人ドロップ → 新しいペアリング行が作成、その選手が片方に入る
- もう片方は空欄状態。別の待機選手をドロップして埋める
- 片方だけの行をキャンセルする場合 → その選手を待機リストにドラッグで戻す → 行が自動削除される

**バリデーション**
- 片方が空欄のペアリング行がある場合、保存ボタンを無効化する（2人揃うまで保存不可）

#### ビジュアルフィードバック
- ドラッグ中の選手カードが半透明で指/カーソルに追従する（DragOverlay）
- ドロップ先のハイライトは不要

#### 操作モード
- ドラッグ&ドロップは編集モード時のみ有効
- 閲覧モード（保存後）ではドラッグ不可。「編集」ボタン押下後に操作可能

#### 待機リスト
- 待機リスト内の並び順は段位順ソート（現状維持）
- 待機リスト内での並び替えドラッグは不要

### 3.2 ビジネスルール
- 対戦履歴表示（日付 or「初」）は現状維持
- 自動マッチング機能は現状維持
- 保存は一括保存（`createBatch` API）で現状維持
- データ構造（`pairings`, `waitingPlayers`）は変更なし

### 3.3 廃止するUI要素
- ドロップダウン（`<select>`）による選手入れ替え → 完全廃止
- 各ペアリング行の右端のゴミ箱ボタン → 廃止
- 「組み合わせを追加」ボタン → 廃止（新規作成ドロップゾーンに置き換え）

### 3.4 エラーケース
- 片方空欄のペアリングがある状態で保存 → 保存ボタン無効化で防止
- ドラッグ中にドロップ可能領域外でリリース → 何も起きない（元の位置に戻る）

## 4. 技術設計

### 4.1 API設計
- バックエンド変更なし
- 既存の `createBatch` API をそのまま使用

### 4.2 DB設計
- 変更なし

### 4.3 フロントエンド設計

#### ライブラリ
- `@dnd-kit/core` — DndContext, DragOverlay, useDraggable, useDroppable, センサー
- `@dnd-kit/utilities` — CSS Transform ユーティリティ

#### 新規コンポーネント（`karuta-tracker-ui/src/pages/pairings/` 配下）

**`DraggablePlayerChip.jsx`**
- `useDraggable` フックでラップした選手カード
- 内部で既存の `PlayerChip` コンポーネントを使用
- ドラッグ中は元の位置が薄く表示される
- `data` プロパティに `playerId`, `playerName`, `source`（どのペアリングの何番目か / 待機リストか）を持たせる

**`DroppableSlot.jsx`**
- `useDroppable` フックでラップしたドロップ受け付け領域
- ペアリング枠（player1 / player2）と新規作成ゾーンの両方に使用
- `data` プロパティに `slotType`（"pairing-player1", "pairing-player2", "new-pairing", "waiting-list"）と `pairingIndex` を持たせる

#### PairingGenerator.jsx の変更

**追加:**
- `DndContext` でペアリングエリア + 待機リストエリア全体をラップ
- `PointerSensor`（PC用）+ `TouchSensor`（スマホ用、`delay` 設定で長押し対応）のセンサー設定
- `DragOverlay` で半透明の追従カード表示
- `onDragEnd` ハンドラ — 既存の `handleSwapPlayer` ロジックを流用してスワップ処理
- 新規作成ドロップゾーンのUI（組み合わせリスト下部）
- 片方空欄ペアリングの保存ボタン無効化ロジック

**削除:**
- 編集モード時の `<select>` ドロップダウン（2箇所）
- 各ペアリング行のゴミ箱ボタン（`handleRemovePair` の呼び出し箇所）
- 「組み合わせを追加」ボタン（`handleAddPairing` の呼び出し箇所）

**変更なし:**
- 閲覧モードの表示
- `handleSave`, `saveDraft` のロジック
- 自動マッチング関連
- 対戦履歴表示

### 4.4 バックエンド設計
- 変更なし

## 5. 影響範囲

### 変更対象ファイル
| ファイル | 変更内容 |
|---------|---------|
| `karuta-tracker-ui/package.json` | `@dnd-kit/core`, `@dnd-kit/utilities` パッケージ追加 |
| `karuta-tracker-ui/src/pages/pairings/PairingGenerator.jsx` | ドロップダウン→D&D置換、削除ボタン・追加ボタン廃止、DndContext統合 |

### 新規作成ファイル
| ファイル | 内容 |
|---------|------|
| `karuta-tracker-ui/src/pages/pairings/DraggablePlayerChip.jsx` | ドラッグ可能な選手カード |
| `karuta-tracker-ui/src/pages/pairings/DroppableSlot.jsx` | ドロップ受け付け領域 |

### 影響を受けないもの
- バックエンド全般（API・DB・DTO）— 変更なし
- `PlayerChip` コンポーネント — 変更なし（`DraggablePlayerChip` が内部利用するのみ）
- `PairingSummary.jsx` — 変更なし（閲覧専用）
- 他画面 — 影響なし

### リスク評価
- **低リスク:** データ構造（`pairings`, `waitingPlayers`）が変わらないため、保存フロー・ドラフト保存との整合性リスクは低い
- **注意点:** `PairingGenerator.jsx` の編集モード描画部分を大きく書き換えるため、既存の状態管理との整合性を注意深く確認する必要がある

## 6. 設計判断の根拠

| 判断 | 理由 |
|------|------|
| `@dnd-kit` を採用 | 軽量・モダン・タッチ対応充実。`react-beautiful-dnd` はメンテ終了気味 |
| バックエンド変更なし | D&DはUI操作の変更のみ。データ構造・保存フローは現状維持 |
| コンポーネントは `pages/pairings/` に配置 | pairings 画面専用の機能であり、他画面での再利用予定なし |
| ドロップダウン完全廃止 | ユーザー要望。D&Dと併用すると操作が混乱する |
| 削除ボタン廃止 | D&D操作との誤タップ防止。待機リストへのドラッグで代替可能 |
| 「組み合わせを追加」ボタン廃止 | 新規作成ドロップゾーンで代替 |
