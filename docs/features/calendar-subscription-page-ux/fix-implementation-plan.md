### タスク1: カレンダー購読画面の UX改善 実装
- [x] 完了
- **対応Issue:** #717
- **親Issue:** #716
- **概要:** `CalendarSubscriptionPage.jsx` を改修し、画面冒頭の目的説明ボックス・カレンダーアプリ登録手順アコーディオン・各操作のサブテキスト強化を一括で実装する。1ファイルに閉じる変更のため1タスクにまとめる。
- **変更対象ファイル:**
  - `karuta-tracker-ui/src/pages/CalendarSubscriptionPage.jsx`
    - 「このページについて」常時表示ボックスを新設し、既存の「所属団体ごと（＋ゲスト参加）にURLが分かれています…」ボックスはこのボックスに統合（重複表示を避ける）
    - `useState` で `openAccordionKey: 'google' | 'apple' | null` を追加（初期値 `null`、同時に1つだけ開く運用）
    - ファイル内に小規模な `AccordionItem` 関数コンポーネントを定義（タイトル、`isOpen`、`onToggle`、`children` を受け取り、`ChevronDown` / `ChevronUp` で開閉表示）。汎用化はしない
    - 「登録手順を見る」セクションを追加し、`AccordionItem` で「Google カレンダー（PCブラウザ）」「Apple カレンダー（iPhone）」の2項目を表示。手順は要件定義書 2.2 の文言素案を採用
    - 各フィードURLカード（`renderFeedCard`）のURL入力欄の下に小サブテキスト「このURLをコピーして、上の登録手順に従ってカレンダーアプリに貼り付けてください」を追加
    - 表示名カスタマイズセクションの説明文を補強し、カレンダー上の表示例（`{表示名}＠{会場名}`、例: 「わすら＠すずらん」）を明示
    - `lucide-react` から `ChevronDown` / `ChevronUp` を追加 import
- **依存タスク:** なし
- **動作確認観点（受け入れ条件）:**
  - 画面冒頭の目的説明ボックスが常時表示されること
  - 「登録手順を見る」アコーディオンが初期状態で全て閉じていること
  - Google / Apple のいずれかを開くと、もう一方は閉じること（同時に1つだけ開く）
  - フィードURL入力欄の下にサブテキストが表示されること
  - 表示名カスタマイズセクションに `{表示名}＠{会場名}` の例示があること
  - 既存のコピー・表示名保存・URL再発行操作が引き続き動作すること
  - レスポンシブ表示（スマホ幅でも読みやすい）
