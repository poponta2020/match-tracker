# カレンダー購読画面 UX改善 改修要件定義書

## 1. 改修概要
- **対象機能:** カレンダー購読画面（`karuta-tracker-ui/src/pages/CalendarSubscriptionPage.jsx`、ルート `/settings/calendar`）
- **改修の背景:** ユーザー要望「この画面は何のためにあって、どうすれば何ができるのか、このページを見れば手順までわかるようにしたい」。現状はフィードURLと操作ボタンが並んでいるだけで、初見ユーザーが画面の目的・カレンダーアプリへの登録手順を画面内で把握できない。
- **改修スコープ:**
  1. 画面冒頭に「この画面の目的」セクションを追加（常時表示）
  2. カレンダーアプリへの登録手順を画面内に表示（Google PCブラウザ / Apple iPhone、アコーディオン形式、テキストのみ）
  3. 各操作の説明を強化（コピーボタン・表示名のサブテキスト常時表示）
- **対象外:**
  - Apple Mac / Outlook の登録手順
  - スクリーンショット・GIFの追加
  - 外部ドキュメントへの誘導リンク
  - バックエンドAPI・DBスキーマの変更（画面内コピーと表示変更のみ）

## 2. 改修内容

### 2.1 項目1: 画面冒頭に「目的説明ボックス」を常時表示

**現状の問題:**
- ページタイトル「カレンダー購読」の直下にいきなりフィードURLカードが表示され、画面の目的・仕組みが説明されていない（現状の説明ボックスは「所属団体ごとにURLが分かれている」点のみ）
- 「一度登録すれば自動更新される」というiCalフィード方式の核心が伝わらない
- 表示対象（キャンセル済み除外、今日以降の参加練習のみ）が画面に明記されていない

**修正方針:**
- タイトル直下に「このページについて」ボックスを常時表示で新設
- 既存の「所属団体ごと（＋ゲスト参加）にURLが分かれています…」の説明ボックスは「目的説明ボックス」に統合（重複を避ける）
- 4つの要素を盛り込む:
  1. 何をする画面か（自動表示の仕組み）
  2. どうやって使うか（利用の流れの3ステップ要約）
  3. 一度登録すれば自動更新というメリット
  4. 参加予定の練習のみ表示というスコープ

**文言素案:**
```
[このページについて]
あなたの参加予定の練習を、Google カレンダーや iPhone のカレンダーに自動表示するためのURLを発行する画面です。

【使い方（3ステップ）】
1. 下のフィードURLをコピー
2. お使いのカレンダーアプリで「URLで購読」を選んで貼り付け
3. 以降はカレンダーアプリが数時間ごとに自動更新します（毎回の操作は不要）

【カレンダーに表示されるもの】
今日以降のあなたの参加予定の練習のみ（キャンセル済みは表示されません）。所属団体ごと＋ゲスト参加で別URLになっており、別カレンダーとして登録すると団体ごとに色分けできます。
```

**修正後のあるべき姿:**
- 初見でこの画面が何のためにあり、何ができるかが、画面内のテキストだけで理解できる

### 2.2 項目2: カレンダーアプリへの登録手順をアコーディオンで画面内表示

**現状の問題:**
- 「Google カレンダー等で各URLを別カレンダーとして登録すると」と書かれているが、具体的にどう登録するかは説明されていない
- 初心者がGoogleやAppleで「URLで追加」「照会するカレンダーを追加」のメニューを探せず詰まる

**修正方針:**
- フィードURLカードの後に「登録手順を見る」アコーディオンセクションを追加
- 初期状態は全て閉じる（折りたたみ済み）。スマホでも縦が長くなりすぎない
- 2つのアコーディオン項目: 「Google カレンダー（PCブラウザ）」「Apple カレンダー（iPhone）」
- 各項目内は番号付きステップのテキスト手順のみ（画像なし）

**文言素案:**

**Google カレンダー（PCブラウザ）:**
```
1. PCのブラウザで calendar.google.com を開きます
2. 左サイドメニュー「他のカレンダー」の右にある「+」をクリック
3. 「URLで追加」を選択
4. このページでコピーしたURLを貼り付け、「カレンダーを追加」をクリック
※ スマホアプリのGoogleカレンダーからは追加できません。PCブラウザで一度追加すれば、スマホのGoogleカレンダーアプリにも自動的に表示されます。
```

**Apple カレンダー（iPhone）:**
```
1. iPhoneの「設定」アプリを開きます
2. 「カレンダー」→「アカウント」→「アカウントを追加」をタップ
3. 「その他」→「照会するカレンダーを追加」をタップ
4. このページでコピーしたURLを貼り付け、「次へ」→「保存」
5. ホーム画面の「カレンダー」アプリで購読カレンダーが表示されます
```

**修正後のあるべき姿:**
- 該当アプリの手順を1クリックで開き、画面のすぐ近くで手順を確認しながら登録できる

### 2.3 項目3: 各操作のサブテキストによる説明強化

**現状の問題:**
- コピーボタンは「コピー」とのtitle属性のみで、コピーした後どうすればよいかが分からない
- 表示名カスタマイズの説明は「設定するとカレンダー上の表示が変わります（例: 早稲田カルタ会 → わすら）」となっているが、実際にカレンダー上でどう見えるか（`{表示名}＠{会場名}` 形式）の例示がない

**修正方針:**
- 各フィードURLカードのURL入力欄の下に小さなサブテキストを追加: 「このURLをコピーして、上の登録手順に従ってカレンダーアプリに貼り付けてください」
- 表示名カスタマイズセクションの説明文を補強し、カレンダー上の表示例（`{表示名}＠{会場名}`）を明示

**修正後のあるべき姿:**
- 各操作ごとに「これは何をする操作か」「結果として何が起こるか」が要素のすぐ下で読み取れる

**JSXの構造変更（上から順）:**

8. URL再発行ボタン（変更なし）

**新規・変更コンポーネント:**
- アコーディオン用の小さな内部コンポーネント `<AccordionItem title onClick isOpen>{children}</AccordionItem>` をファイル内に定義（汎用化はしない、YAGNI）
- アコーディオンの開閉アイコンは `lucide-react` の `ChevronDown` / `ChevronUp` を使用

**state追加:**
- `openAccordionKey: 'google' | 'apple' | null`（同時に1つだけ開ける運用、シンプル化）

**スタイル方針:**
- 既存の `bg-white rounded-lg border p-4` を踏襲し、視覚的に統一
- 「このページについて」ボックスは情報密度が高いため `bg-blue-50` または `bg-gray-50` でわずかに目立たせる（最終色は実装時に既存トーンと合わせて調整）

## 5. 設計判断の根拠

### 5.1 なぜ画面冒頭に「目的説明ボックス」を常時表示するか
- 初見ユーザーが必ず読むべき内容だから（折りたたむと読まない）
- 一度設定したユーザーには冗長だが、設定変更などで稀に訪れる際にも害はない（短く要点を絞れば視覚ノイズは小さい）

### 5.2 なぜカレンダー登録手順をアコーディオンにするか
- 全アプリを常時展開すると画面が長くなり、URL本体が画面外に押し出される
- タブ切替よりアコーディオンの方が、一度に複数アプリの手順を確認しやすい（スクロールで比較できる）

### 5.3 なぜ Google PCブラウザと iPhone Appleカレンダーだけか
- 本サービスのメインユーザーはこの2系統で大半をカバーできるとユーザーが判断
- Mac版Apple / Outlook は要望が出てきた時点で追加すればよい（YAGNI）

### 5.4 なぜテキスト手順のみで画像を入れないか
- スクリーンショットはGoogle / Apple のUI更新で陳腐化しやすく、メンテコストが大きい
- 画像ファイル管理（保存先・ビルドへの取り込み）の追加要素が増える
- 番号付きステップ＋メニュー名で十分伝わる

### 5.5 なぜ説明をサブテキスト常時表示にするか（ヘルプアイコン式ではなく）
- スマホで ? アイコンをタップしないと説明が見られないのは、初見ユーザーには発見されにくい
- 短いサブテキストなら視覚的にもうるさくない

### 5.6 なぜ Apple アコーディオンを「設定アプリから追加する手順」にするか
- iPhoneの「カレンダー」アプリ内からも追加できるが、UI階層が深く分かりにくい
- 設定アプリ経由のフローは安定しており iOS バージョンが上がっても変わりにくい
