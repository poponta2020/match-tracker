# match-list-column-alignment 実装手順書

## 実装タスク

### タスク1: MatchList 行レイアウトの CSS Grid 化と列順変更
- [x] 完了
- **概要:**
  - 対戦履歴一覧の各行を `flex` から CSS Grid によるテーブル風レイアウトに書き換える。
  - 6 列構造 `[日付] [対戦相手名] [勝敗] [会場 N試合目] [メモアイコン] [お手付き]` で、各列の左端 x 座標が全行で揃うようにする。
  - 名前列・会場列は超過時に `truncate`（`…` で省略）。
  - メモアイコン・お手付きは条件で非表示になる場合でも列幅は確保し続ける（行間で後続列の x 座標がズレないようにするため）。
  - 既存の `formatDate()` / `getResultDisplay()` / `getResultColor()` ユーティリティ、対戦相手名タップ動線、メモアイコンタップ動線、ゲスト選手のリンク無効化、メンター関係 ACTIVE 時のみメモアイコン表示、メモ有無による色分けは全て維持する。
- **変更対象ファイル:**
  - `karuta-tracker-ui/src/pages/matches/MatchList.jsx` — 試合一覧の行 JSX（535-580行付近）を CSS Grid ベースに書き換え。列順を新しい順に変更。列幅は Tailwind の `grid grid-cols-[...]` で固定値を持たせる（具体値は実装者裁量だが、モバイル幅で 6 列が無理なく収まること）。
- **依存タスク:** なし
- **対応Issue:** #762
