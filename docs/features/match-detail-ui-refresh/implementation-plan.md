# 試合詳細画面 UI リファイン 実装手順書

## 実装タスク

### タスク1: 試合結果・詳細情報・メモを1つのカードに統合し、コンパクト表示にする
- [x] 完了
- **概要:**
  - 現状の3つのカード（試合結果カード / 詳細情報カード / メンティーのメモカード）を1つに統合する
  - 試合結果サマリを `{対戦相手名}  {○/×/△} {枚数差絶対値}` の1行コンパクト表示に変更
  - 詳細情報を `{試合日}  {第N試合}  {会場名}`（半角スペース2個区切り、アイコンなし）の1行表示に変更
  - メンター閲覧時の「メンティーのメモ」別カードを廃止し、統合カードの下段に表示
  - カードフッターの「作成日 / 更新日」を削除
  - 不要になった `lucide-react` の import（`Trophy`、`User`、`Calendar`、`MapPin`）を削除
  - 勝敗マーク変換ヘルパー（`getResultMark`）を追加し、勝ち→`○` / 負け→`×` / 引き分け→`△` で変換
  - 対戦相手名タップで試合一覧へ遷移する現状機能は維持
- **変更対象ファイル:**
  - `karuta-tracker-ui/src/pages/matches/MatchDetail.jsx` — カード統合、表示形式変更、不要要素削除
- **依存タスク:** なし
- **対応Issue:** #815

### タスク2: メンティー本人画面でのコメント欄表示制御を追加
- [x] 完了
- **概要:**
  - メンティー本人画面（`isOtherPlayer=false`）で、自分以外の投稿者によるコメントが1件以上ある場合のみ `MatchCommentThread` を表示するロジックを追加
  - メンター閲覧時（`isOtherPlayer=true`）は現状の表示ロジックを維持（メンター関係があれば表示）
  - 判定は `MatchDetail` 内で `matchCommentsAPI.getComments` を呼んで取得したコメント一覧の `authorId !== currentPlayer.id` で行う
  - `MatchCommentThread.jsx` 自体には変更を加えない（内部の `fetchComments` で再フェッチして投稿後の更新を行うため）
- **変更対象ファイル:**
  - `karuta-tracker-ui/src/pages/matches/MatchDetail.jsx` — `commentsByOthersExist` ステートと `useEffect` 追加、`MatchCommentThread` の表示条件変更、`matchCommentsAPI` の import 追加
- **依存タスク:** なし（タスク1とは独立してもよいが、同じファイルを編集するためタスク1と同じPRで進めるのが望ましい）
- **対応Issue:** #816

### タスク3: ドキュメント更新
- [x] 完了
- **概要:**
  - `docs/SPECIFICATION.md` / `docs/SCREEN_LIST.md` / `docs/DESIGN.md` のうち、試合詳細画面の表示に関する記述があるものを更新
  - 試合結果カード / 詳細情報カードの統合、コメント欄の表示条件変更を反映
- **変更対象ファイル:**
  - `docs/SPECIFICATION.md` — 試合詳細画面のレイアウト・コメント欄表示条件の記述更新（該当箇所がある場合）
  - `docs/SCREEN_LIST.md` — 試合詳細画面の説明更新（該当箇所がある場合）
  - `docs/DESIGN.md` — 該当箇所がある場合のみ更新
- **依存タスク:** タスク1、タスク2
- **対応Issue:** #817
