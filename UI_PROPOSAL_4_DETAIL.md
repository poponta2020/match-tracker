## 色分けルール

### 少ない（1-3名）- 赤系バッジ
```
┌───────┐
│   3   │  ← 背景: #FEE2E2 (赤50)
└───────┘     文字: #991B1B (赤800)
```
**意味**: 参加者が少ないので参加推奨

### 普通（4-6名）- グレー系バッジ
```
┌───────┐
│   5   │  ← 背景: #E5E7EB (グレー200)
└───────┘     文字: #374151 (グレー700)
```
**意味**: 適度な人数

### 多い（7名以上）- 緑系バッジ
```
┌───────┐
│   8   │  ← 背景: #D1FAE5 (緑100)
└───────┘     文字: #065F46 (緑800)
```
**意味**: 参加者が多いので調整が必要な場合も

### 0名 - 薄いグレー
```
┌───────┐
│   0   │  ← 背景: #F3F4F6 (グレー100)
└───────┘     文字: #9CA3AF (グレー400)
```
**意味**: まだ誰も参加していない

## レスポンシブ対応

### PC表示
- 7試合すべて横並び
- バッジサイズ: 32px × 20px
- フォント: 11px

### タブレット（768px以下）
- 横スクロール可能
- バッジサイズそのまま

### スマホ（640px以下）
- 横スクロール必須
- または試合列を折り返し表示（別デザイン案）

## 実装コード例

```jsx
<td className="px-2 py-3 text-center">
  <div className="flex flex-col items-center gap-1">
    {/* チェックボックス */}
    <input
      type="checkbox"
      checked={isChecked}
      onChange={() => toggleMatch(session.id, matchNumber)}
      disabled={isPast}
      className="w-5 h-5 text-primary-600 border-gray-300 rounded
                 focus:ring-primary-500 disabled:opacity-50"
    />

    {/* 参加人数バッジ */}
    <span
      className={`
        text-xs px-2 py-0.5 rounded font-medium
        ${participantCount <= 3
          ? 'bg-red-50 text-red-800'
          : participantCount >= 7
          ? 'bg-green-100 text-green-800'
          : 'bg-gray-200 text-gray-700'}
      `}
      title={`試合${matchNumber}に${participantCount}名参加中`}
    >
      {participantCount}
    </span>
  </div>
</td>
```

## メリット

1. **一目で人数が分かる**: スクロールせずに全試合の参加状況を把握
2. **色で混雑度が分かる**: 赤=参加推奨、緑=満員に近い
3. **コンパクト**: 既存デザインを大きく崩さない
4. **直感的**: チェックボックスのすぐ下に関連情報
5. **拡張可能**: ホバーで詳細表示も可能

## デメリットと対策

**デメリット1**: セルの高さが増える
→ 対策: パディングを調整してコンパクトに

**デメリット2**: 色覚異常への配慮
→ 対策: 色だけでなく数字でも判断可能

**デメリット3**: スマホで見づらい可能性
→ 対策: 横スクロール + タッチ操作最適化
